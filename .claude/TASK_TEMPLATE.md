# TASK_TEMPLATE — 统一任务模板

> 复制对应类型模板，填写后放入 `.claude/tasks/active/`

---

## 模板 A：复杂开发

```markdown
# [任务标题]

## 类型
复杂开发

## 背景
[一句话：为什么要做，解决什么问题]

## 目标
### P0（必须）
- [ ] [可验收目标]

### P1（尽量）
- [ ] [可选目标]

## 涉及 Skill
- lds-safe-backend（若涉及 server/ 或 data/）
- lds-project-quotation（若涉及报价结构）
- lds-frontend-ui（若涉及页面）

## 关键约束
- 零外部依赖（不引入新 npm 包）
- npm test 必须通过
- [其他约束]

## Done Definition
- [ ] [具体可验证标准 1]
- [ ] [具体可验证标准 2]
- [ ] npm test 全部通过
- [ ] 交付说明完整

## 验收方式
[操作步骤，可直接执行]

## 风险提示
| 风险 | 概率 | 影响 | 处置 |
|---|---|---|---|
| [风险描述] | 高/中/低 | [影响] | [处置] |

## 交付说明（完成后填写）
- 改动文件：
- 数据改动：
- 兼容处理：
- 本地验证：
- 线上风险点：
- 本地 server / 线上 api 同步状态：
- 下一轮可打包项：
```

```markdown

[web/xxx.html]

## 问题描述
[视觉/交互问题，尽量附截图路径]

## 预期效果
[改后是什么样]

- 仅样式（颜色/间距）→ Codex 可
- 涉及事件绑定/逻辑 → Claude Code

- 修改 web/styles.css 而非内联 style
- 新增文字加入 web/ui-labels.js
- 不引入外部 CSS 框架
```
