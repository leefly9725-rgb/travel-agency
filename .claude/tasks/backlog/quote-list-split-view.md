# 报价列表拆分视图 — 标准报价 vs 项目型报价

## 类型
架构讨论 → 前端改动

## 状态
backlog — 当混合列表变得难以维护时启动

## 触发条件（满足其一即可启动）
- project 型报价数量超过 30 条
- 用户明确反馈"标准报价和项目报价要分开看"
- `/api/quotes` 列表的筛选维度超过 3 个

## 方案选项（需讨论）
- Tab 切换（同页面，改动小）
- 独立页面（`/project-quotes.html`）
- 统一列表 + 类型标签过滤

## 注意
改动涉及 `/api/quotes` 路由（可能需要 `pricingMode` 过滤参数），属于 lds-safe-backend 范畴。
